# U64x32 Encoding

`U64x32` is a compact **binary** pseudo-floating number format used to store spot prices inside compressed oracle slots. It packs a non-negative integer into 32 bits by storing a 27-bit mantissa and a 5-bit exponent.

## Layout

```
bits 31 … 27 : exponent (5 bits)
bits 26 …  0 : mantissa (27 bits)
```

- Mantissa ∈ `[0, 2^27 - 1]`.
- Exponent ∈ `[0, 31]`.
- Decoded value: `value = mantissa << exponent`.

Maximum representable value is `(2^27 - 1) << 31`.

## Encoding Strategy

This repository only includes a decoder on-chain; encoding is expected to happen off-chain.

When encoding an integer price off-chain:

1. Choose `exponent` so that `mantissa = value >> exponent` fits into 27 bits.
2. Pack `packed = (exponent << 27) | mantissa`.
3. Decoding will round down to the nearest representable value.

## Typical Error Bounds

- Precision is best for smaller exponents; larger exponents trade precision for range.
- Downstream systems are responsible for applying any fixed-point scaling (e.g., interpreting the decoded integer as `1e8`-scaled).

## Usage

- `U64x32.decode(uint32)` — expands the stored 32-bit word back to a 64-bit integer (rounding down).

The oracle stores the `U64x32` word and leaves final scaling (e.g., to 1e18) to downstream systems.
