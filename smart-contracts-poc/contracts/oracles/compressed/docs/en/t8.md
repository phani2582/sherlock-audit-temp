# T8 Time Encoding

`T8` compresses Unix timestamps into 8-bit “ticks” and implements RFC‑1982 style wrap-around comparison. It is used by `CompressedOracleV1` to track freshness without storing full 256-bit block numbers.

## Parameters

- `TICK = 1 second` → every tick is a one-second increment.
- `HALF = 128` → wrap detection window; values differ by less than 128 ticks are considered ordered.
- Encoded tick: `uint8(now / TICK)`.

## Helpers

- `encode(uint256 unix)` → returns `uint8` tick.
- `getTicks(uint256 duration)` → ceiling division, useful for converting seconds into ticks.
- `deadlineFromNow(now, duration)` → future tick respecting wrap semantics.
- `notAfter(deadline, now)` → “deadline ≥ now” taking wrap-around into account.
- `strictlyBefore(deadline, now)` → true when now falls strictly before deadline in the cyclic space.

## Why 8 Bits?

- Short-term freshness checks: the oracle invalidates feeds after at most 127 seconds (`HALF - 1`).
- Minimal storage: timestamp lives in the lowest byte of the compressed slot.
- RFC-1982 arithmetic ensures comparisons remain correct even after tick wraps from `255 → 0`.

## Best Practices

- Treat `0xFF` as the sentinel “no data / invalidated” tick; on creation or stale eviction the slot stores this value.
- When comparing two ticks, use `notAfter` or replicate its logic; naive `>` comparisons break across wraps.
- Keep `maxStaleSeconds < HALF` to avoid ambiguous comparisons; the contract enforces this in `setMaxStaleSeconds`.
