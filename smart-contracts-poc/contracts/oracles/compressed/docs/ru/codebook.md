# Codebook256: справочник индексов

`Codebook256` — таблица соответствия из 8-битного индекса (0–255) в 16-битное значение `[0, 10_000]`. Она распределяет значения неравномерно: вокруг нуля плотность высокая, а в верхней части диапазона шаги шире. Благодаря этому мы можем хранить спред и волатильность в одном байте без сильной потери точности.

## Формат

- На смарт-контракте таблица лежит как `bytes constant TABLE`; каждая запись — два байта (big-endian).
- Декодирование: `value = TABLE[i*2] << 8 | TABLE[i*2 + 1]`.
- Генерация происходит оффчейн и «вшивается» в контракт перед сборкой.

## Цели распределения

- Макс. разрешение около небольших значений (0–50 bp).
- Допускаем крупные шаги в большом диапазоне (до 10_000), чтобы уложиться в 256 позиций.
- Запрещаем дубликаты, чтобы отображение оставалось детерминированным (обратный поиск делается оффчейн).

## Как перегенерировать таблицу

Исходники лежат в `codebook/*.json`. После правок выполните:

```bash
npm run codebook:generate
```

Скрипт:

1. Собирает все числа из JSON в один байтовый массив.
2. Валидирует границы и отсутствие повторов.
3. Перезаписывает `contracts/oracles/utils/Codebook256.sol`, обновляя константу `TABLE`.

Не забывайте коммитить и JSON, и обновлённый Solidity-файл, чтобы сборка была воспроизводимой.

## Практические заметки

- Обратное отображение (value → index) делайте оффчейн: линейный поиск в Solidity слишком дорог.
- При изменении распределения обновляйте соответствующие тесты — допуски по точности могут поменяться.
- Таблица ограничена 256 элементами; превышение приведёт к ошибке генерации.
