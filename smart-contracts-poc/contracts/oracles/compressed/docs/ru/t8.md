# Временное кодирование T8

`T8` — 8-битное кодирование времени с семантикой RFC-1982. Оно позволяет хранить «тик» (секунду) в одном байте и корректно сравнивать значения после переполнения.

## Параметры

- `TICK = 1 секунда` — длина одного шага.
- `HALF = 128` — половина кольца (используется для проверки «свежее/протухло»).
- Кодирование: `encode(now) = uint8(now / TICK)`.

## Вспомогательные функции

- `encode(unix)` — переводит секунды Unix в тик.
- `getTicks(duration)` — потолок от `duration / TICK`.
- `deadlineFromNow(now, duration)` — будущее значение с учётом wrap-around.
- `notAfter(deadline, now)` — true, если `deadline` не раньше `now` в циклическом пространстве.
- `strictlyBefore(deadline, now)` — строгая версия «раньше» (исключает равенство).

## Почему 8 бит достаточно

- Следим только за «свежестью» фида на горизонте < 128 секунд.
- Экономим место: тик находится в младшем байте storage-слота.
- RFC-1982 гарантирует, что сравнения остаются корректными даже после переполнения `255 → 0`.

## Практические заметки

- `0xFF` используется как «sentinel»: нет данных или фид признан устаревшим (`invalidateFeeds`).
- При сравнении двух тиков всегда используйте `notAfter` или аналог — простой `>` ломается на переполнении.
- Ограничение `maxStaleSeconds < HALF` исключает неоднозначность; контракт применяет `require` в `setMaxStaleSeconds`.
