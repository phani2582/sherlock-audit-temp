# Кодирование U64x32

`U64x32` — компактный **двоичный** формат «псевдоплавающих» чисел для хранения цен в сжатых слотах. Он упаковывает неотрицательное целое в 32 бита, используя 27-битную мантиссу и 5-битный показатель степени.

## Структура

```
биты 31 … 27 : показатель степени (5 бит)
биты 26 …  0 : мантисса (27 бит)
```

- Мантисса ∈ `[0, 2^27 - 1]`.
- Показатель степени ∈ `[0, 31]`.
- Декодированное значение: `value = mantissa << exponent`.

Максимально представимое значение: `(2^27 - 1) << 31`.

## Как кодировать

On-chain в репозитории есть только декодер; кодирование предполагается off-chain.

Примерная стратегия кодирования:

1. Выбираем `exponent` так, чтобы `mantissa = value >> exponent` помещалась в 27 бит.
2. Собираем `packed = (exponent << 27) | mantissa`.
3. Декодирование округляет вниз до ближайшего представимого значения.

## Точность

- Точность выше при малых `exponent`; при росте `exponent` увеличивается диапазон, но падает точность.
- Финальный scale (например, трактовать декодированное значение как `1e8`-scaled) задаётся downstream-системами.

- `U64x32.decode` — разворачивает 32-битное слово обратно в `uint64` (округляя вниз).

Внутри `CompressedOracle` хранится именно кодированное значение, а финальное приведение масштаба (до 1e18 и т.п.) выполняют downstream-системы.
